    How To Make a Pull Request and Merge on Github


    In this section I will show you how make a pull request from webstorm and merge it into the repository on Github. First lets define some terms that we will be using.


    Push: Pushing will commit your changes to the repository on github. You would do this when you want to add you code to the master code in git.


    Pull: A pull is when you taking in the changes made and merging them with your master.


    Pull Request: A Pull Request would be sent for the admin of the repository to be checked before it is merged into the master. This is helpful so the admin can determine if your changes wont mess up the master file. If the admin accepts your pull request and merges it, your code becomes a permanent part of the code.


    Now that youve finished making the changes in your new branch you're probably wondering how your getting it into Github. I made a branch called, "DefinePull". I go to the bottom right click the branch and select, "Push".



    A box will pop up with the changes you made. On the bottom right of the box is a button called, "Push", click it.



    Now go to your github repository and you'll see your sent in branches. In my case its my definePull branch. You want to click, "Compare & Pull request".



    Another box will pop up. Here you could leave comments and also create the pull request.



    In this screen the code will be analyzed but GitHub to determine if the merge is ok. If it is you'll be able to click the button, "Merge Pull request".



    Congrats your code from webstorm has been merged perfectly into you repository on GitHub!
