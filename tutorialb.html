    Webstorm and AFS

    For this following tutorial, I will show you how to set-up AFS on Webstorm. How to connect it with your school AFS .


    For the first step you must go to Tools like we did in the previous tutorial. Then click on Start SSH session...


    After doing that a window will appear at the middle of the screen. This is where you have to enter the college AFS url, username and password.

    Once you filled everything with the correct information you will be able to log in to the college AFS successfully. However, the site will not be working as more steps will be needed.


    For the next steps we will follow the first tutorial. Going into Tools then to Deployment. You will see an option called Configuration.... This is where we will set-up AFS to work for your site. We will have to click on the + icon and create a new SFTP configuration.

    Once you create a new SFTP configuration then you will have to enter the same information as before from Start SSH session.

    The only changes we have to make is for the Root path and Web service URL. THe only thing you do is to click on Autodetect. Then simply add /public_html next to your ID. The following step is to change a couple of things from the Web service URL. Instead of afs13 we just change that to web. Then we just add /~your_ID to the end of the url.


    After configuring everything and if you can successfully connect then we can proceed to the next step. If you are getting error when connecting, your host name, username or password might be wrong. As for the following step we will proceed to Mappings next to Connection in the above tabs.

    These are simpler steps if you are not working with a HTML5 Boilerplate. First, we just copy the name of your project in to the followings. For example, my project name is MajorProject1. I will use that same name and paste it into Deployment path and Web path. For the latter, we simply add a / at the beginning.


    If you are intending on using the HTML5 Boilerplate then we must do a couple of different steps. First, we must add the HTML5 Boilerplate url to the remote. Although, you must Enable Version Control Integration. That will allow you to connect to your Github account. After doing that you can locate Remotes by going to the VCS tab. Then Git then Remotes....



    Proceeding with the Mappings we must click on the folder icon as you can see below.

    Once you do that a new window will pop up. Here we simply choose the project we are working on and go to the dist(distribution) directory. Select it and click OK.
