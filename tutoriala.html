    Webstorm Tutorial

    Autosave/Auto Upload
    In order for your changes to take effect or automatically upload to AFS we must do the following. First you select the Tools bar.


    Then proceed to select Deployment.


    Then click Automatic Uploads. This will allow Webstorm to automatically upload any changes you make to a file or folder. The changes will we displayed once you open the site without having to manually auto-save it.


    Webstorm Files

    Creating files in Webstorm is easy. However, there are rules you should follow. Whenever you create an .html file, you should be aware that everything must be in lower-cased. The reason for this is because if you are working as a group with someone else and they have to access or use your file, they might encounter a problem. The problem would be that they are entering a lower-cased named file, but your file name is upper-cased. So, it is safe to stay away from capital letters.


    First of all, to create a file you must right click on your project folder and select New then File.


    In this case you have the option to write whichever file type you want. For example, index.html or style.css An easier way is to right click your project then New and choose whichever file you want from the list below. You can also create a folder by right clicking you project and New then Directory.


    To see the directories/files going into the AFS, there is a tab called Remote Host on the right side. If you can't find it or is not showing up for you, then you must go to Tools then Deployment and click on Browse Remote host. The tab and window will appear on the right side.
