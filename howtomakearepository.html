    How To Make Github Repository in Git and Webstorm


    First we should define what a Repository is. A Repository is like a folder for your project. Your project's repository contains all of your project's files and stores each file's revision history. First you click the + Icon on the top right corner of www.github.com and click "New Repository".


    On this page you can select if you want to be the owner or you want an organization should be the owner. Also on this page you can give your repository a name. In the above picture I have assigned this repository to myself and named it, "example_Repository". Then I hit thh "Create Repository" Button.


    In the above picture you'll see that there is a link for the repository, you should save that link for later when we are setting up the repository through Webstorm. If you already had some code you could upload it here or just create an empty repository.


    You could also upload to your Repository right from Webstorm or create a new Repository. You do this first by creating a project. Then you go to VCS in the top menu bar and click it. You will see an option called "Enable Version Control Integration".


    You will then be prompted by a box and in this box click the drop down menu and select Git.


    Navigate back to the menu bar on the top and click "VCS", then click, "Import into Version Control", then click, "Share Project on Github". This will bring up a new window.


    Now all you have to do is give it a name and click share. Now you have created a new repository on github through Webstorm. Congrats!!

    Helpful Terms To Know


    Clone: When you create a repository on GitHub, it exists as a remote repository. You can clone your repository to create a local copy on your computer and sync between the two locations.


    Fork: A fork is a copy of a repository. Forking a repository allows you to freely experiment with changes without affecting the original project.


    Gitignore: A .gitignore is a file you can configure to ignore files you don't want to check in to GitHub.
